# 小说沙盘 V4.2 开发完成

## 版本概述

V4.2 是小说沙盘的增强版本，专注于前端用户体验，提供三个核心生成器和一个统一的审核面板。

### 核心功能

1. **角色生成器** - 灵活配置参数，批量生成角色候选
2. **NPC生成器** - 支持多种类型的NPC生成，包含目击者、阻碍者等
3. **线索生成器** - 支持多条发现路径配置的线索生成
4. **候选审核面板** - 统一管理所有生成的候选，支持批量操作

## 功能详细说明

### 1. 角色生成器 (CharacterGenerator.vue)

**功能特性：**
- 可选择角色类型（主角、配角、反派、NPC）
- 设定生成数量（1-10个）
- 选择剧情线（医院真相、校园怪谈、老宅秘闻等）
- 调整创意参数（0-100）
- 生成的角色包含：
  - 基本信息（姓名、角色类型、角色定位）
  - 性格特点（2-4个特质）
  - 动机目标（短期+长期）
  - 人物背景（简介）
  - 剧情钩子（1-2个与故事主线相关的关联）

**使用流程：**
1. 选择角色类型和剧情线
2. 调整生成参数
3. 点击生成按钮
4. 在候选审核面板中编辑、验证、批准

### 2. NPC生成器 (NPCGenerator.vue)

**功能特性：**
- 支持多种NPC类型：目击者、阻碍者、线人、守门人
- 关联指定地点（医院大厅、院长办公室等）
- 每条线索包含2-4条记忆
- 每条记忆包含对话和获得方式
- 为每个角色添加1-2个可获得的物品

**NPC类型说明：**
- **目击者** - 看到关键事件或人物的角色
- **阻碍者** - 设置障碍、拒绝透露信息的角色
- **线人** - 掌握多条线索的重要角色
- **守门人** - 控制关键地点入口的角色

### 3. 线索生成器 (ClueGenerator.vue)

**功能特性：**
- 支持多种线索类型：物证、痕迹、记录、记忆、声音、时间
- 关联指定地点
- 每条线索生成多条发现路径
- 每条发现路径包含：方式、所需条件、获得线索
- 线索重要程度分级（1-5）
- 支持发现限制（有/无）

### 4. 候选审核面板 (CandidateReviewPanel.vue)

**功能特性：**
- 候选状态统计（待处理、已验证、已批准）
- 支持按类型和状态筛选
- 批量操作（批量验证、批量批准、批量入库）
- 单个候选操作（编辑、验证、批准、拒绝）
- 候选详细信息展示：
  - 基本信息展示
  - 性格特点标签
  - 技能评分展示
  - 发现路径列表
  - 验证报告（完整性、引用检查、知识边界、综合评分）

**候选状态流转：**
```
generated (刚生成) → edited (已编辑) → validated (已验证) → approved (已批准) → committed (已入库)
```

## 技术栈

- **框架**: Vue 3 (Composition API)
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **UI组件库**: PrimeVue
- **样式**: Tailwind CSS

## 核心状态管理 (stores/generator.js)

**Store功能：**
- 候选的CRUD操作
- 批量添加、验证、批准、入库
- 候选状态管理
- 生成日志记录
- 候选类型分组

**主要计算属性：**
```javascript
characterCandidates    // 角色候选
npcCandidates          // NPC候选
locationCandidates     // 地点候选
clueCandidates         // 线索候选
pendingCandidates      // 待处理候选
validatedCandidates    // 已验证候选
approvedCandidates     // 已批准候选
```

## 使用说明

### 1. 启动开发服务器

```bash
cd web
npm install
npm run dev
```

### 2. 使用流程

**步骤一：生成候选**
1. 在左侧导航选择生成器类型（角色、NPC、线索）
2. 配置生成参数
3. 点击「生成候选」按钮
4. 系统将批量生成指定数量的候选

**步骤二：审核候选**
1. 切换到「候选审核面板」
2. 查看所有生成的候选
3. 使用筛选器按类型或状态过滤
4. 点击验证按钮对候选进行验证
5. 验证通过后批准候选

**步骤三：批量入库**
1. 在审核面板选择多个候选（可选）
2. 点击批量验证/批量批准/批量入库按钮
3. 已入库的候选可用于后续的小说生成

### 3. 验证报告说明

验证报告包含四个维度的评分：
- **完整性**: 检查必填字段是否完整
- **引用检查**: 检查引用的角色/地点是否存在
- **知识边界**: 检查内容是否在规定范围内
- **综合评分**: 70-99分的综合质量评分

## 扩展开发

### 添加新的生成器

1. 在 `views/` 目录下创建新的生成器组件
2. 在 `stores/generator.js` 中添加新的候选类型和相关计算属性
3. 在 `router/index.js` 中添加路由配置
4. 在 `App.vue` 中添加导航链接

### 新增候选类型

1. 在 `generator.js` 中的 `getCandidateTypeLabel` 添加新类型标签
2. 在审核面板中添加相应的展示逻辑
3. 如需要特殊验证逻辑，扩展 `validateCandidate` 方法

## 已知限制

1. 当前生成器使用的是模拟数据生成（真实的LLM生成接口尚未接入）
2. 数据仅存储在内存中，刷新页面后丢失
3. 未实现真实的数据持久化存储
