# V3.1 开发计划：ProgressMonitor + Lightweight Director

> 版本主题：解决剧情卡住与无进展问题  
> 目标：在不引入复杂导演系统的前提下，让系统能检测剧情停滞，并通过轻量干预让故事继续推进。

---

## 1. 背景

V2 已经实现：

```text
LLM Agent
多地点探索
轻量记忆
事件日志
章节改写
一致性检查
```

但模拟过程中仍可能出现：

```text
Agent 连续重复 observe / wait
Agent 找不到关键线索
Agent 不知道该去哪个地点
Agent 避开冲突，导致故事变平
章节目标迟迟无法完成
```

V3.1 的作用是：

> 不直接操控角色，而是通过检测停滞并投放轻量提示，让 Agent 自然继续行动。

---

## 2. 版本目标

实现两个核心模块：

```text
ProgressMonitor
LightweightDirector
```

它们要完成：

```text
1. 识别连续无进展
2. 识别重复动作
3. 识别章节目标推进不足
4. 识别当前可用线索入口
5. 生成 soft_hint / environment_hint / npc_pressure
6. 将干预写入 EventLog
7. 将干预注入下一轮 AgentContext
```

---

## 3. 不做什么

V3.1 不做完整 Director 系统。

暂不实现：

```text
复杂张力评分
多维度戏剧节奏控制
复杂事件投放
完整 PlotArc 推理
人物弧光推进
伏笔回收
```

V3.1 只做“剧情卡住时的轻量救场”。

---

## 4. 核心概念

### 4.1 progress event

以下事件可以视为 progress event：

```text
discovery：发现新线索
conflict：产生冲突
relationship_change：关系变化
goal_update：目标变化
new_location：进入新地点
new_topic：开放新话题
chapter_goal_progress：章节目标推进
```

### 4.2 no progress window

如果最近 N 个 tick 没有 progress event，则判定为无进展。

推荐默认：

```text
no_progress_tick_threshold = 5
```

### 4.3 repeated action

如果同一角色连续多次对同一 target 做相同 action，则判定重复。

例如：

```text
char_linzho inspect hospital_gate_lock 连续 3 次
```

推荐默认：

```text
repeat_action_threshold = 3
```

## 6. 模块设计

## 6.1 ProgressMonitor

### 职责

```text
1. 读取最近 N 个 tick 的 EventLog
2. 判断是否有 progress event
3. 判断是否重复动作
4. 判断章节目标是否推进
5. 判断是否需要干预
6. 输出 ProgressReport
```

### 输入

```text
simulation_id
current_tick
recent_events
world_state
chapter_goal_status
```

```text
ProgressReport
```

## 6.2 LightweightDirector

### 职责

```text
1. 读取 ProgressReport
2. 查看当前地点未触发 route
3. 根据当前问题选择干预类型
4. 生成 DirectorProposal
5. 交给 InterventionApplier 写入 EventLog
```

### 干预类型

V3.1 只支持 3 种：

```text
soft_hint：注入 AgentContext 的轻提示
environment_hint：环境出现轻微变化
npc_pressure：NPC 施压或开启新 topic
```

---

## 7. 干预规则

### 7.1 soft_hint

不写入 EventLog，只注入 AgentContext。

适合：

```text
轻微停滞
Agent 重复动作
存在明显未探索 target
```

示例：

```json
{
  "type": "soft_hint",
  "target_agent": "char_linzho",
  "content": "你注意到前台后方似乎还有没有仔细检查过的地方。"
}
```

---

### 7.2 environment_hint

写入 EventLog，改变可见环境。

适合：

```text
连续无进展
当前地点存在未触发 discover_route
需要开放新的 target 或 route
```

要求：

```text
不能直接发现线索
不能直接写入 discovered_facts
只能开放 route 或 target
```

---

### 7.3 npc_pressure

写入 EventLog，改变 NPC 行为或开放话题。

适合：

```text
冲突太低
同地点存在 NPC
当前 NPC 有可问 topic
```

示例：

```json
{
  "type": "npc_pressure",
  "actor": "char_guard",
  "target": "char_linzho",
  "description": "看门人忽然挡在楼梯口，语气比刚才更硬。",
  "available_topics_added": ["why_block_stairs"],
  "relationship_delta": -5
}
```

## 8. AgentContext 增强

V3.1 增加：

```json
{
  "soft_hints": [
    "你注意到前台后方似乎还有没有仔细检查过的地方。"
  ],
  "recent_progress_report": {
    "no_progress_ticks": 5,
    "repeat_actions": []
  }
}
```

注意：

```text
soft_hints 不能写成命令
不能出现“你应该”
不能暴露隐藏事实
```

## 9. Tick Loop 接入

```text
For each tick:
    执行 Agent 行动
    EnvironmentEngine 判定
    写入 EventLog
    更新 WorldState
    ProgressMonitor.evaluate
    If need intervention:
        LightweightDirector.propose
        InterventionApplier.apply
        写入 intervention event
        更新 AgentContext soft_hints
```

## 11. 验收用例

### 用例 1：连续无进展

条件：

```text
最近 5 tick 没有 discovery/conflict/relationship_change
```

预期：

```text
ProgressMonitor 输出 no_progress
LightweightDirector 生成 environment_hint
```

---

### 用例 2：重复动作

条件：

```text
同一角色连续 3 次 inspect 同一 target
```

预期：

```text
ProgressMonitor 标记 repeat_action
AgentContext 注入 soft_hint
```

---

### 用例 3：当前地点存在未触发 route

条件：

```text
hospital_lobby 有 route_hf003_search_front_desk 未触发
```

预期：

```text
environment_hint 解锁或强调 front_desk
但不直接发现 hf_003
```

---

### 用例 4：NPC 施压

条件：

```text
当前地点有 char_guard
conflict_score 低
```

预期：

```text
npc_pressure 事件写入 EventLog
新增 available_topic
relationship 下降
```

---

## 12. DoD

```text
1. 连续 5 tick 无 progress event 时能识别停滞
2. 重复动作能被识别
3. ProgressReport 能写入 progress_reports.jsonl
4. LightweightDirector 能生成 soft_hint / environment_hint / npc_pressure
5. environment_hint 不能直接写入 discovered_facts
6. npc_pressure 能开放新 topic 或改变 relationship
7. 干预事件写入 EventLog
8. AgentContext 能感知 soft_hints
9. 每章干预次数可限制
10. 干预不能暴露 forbidden truth
```
