# V3.5 开发计划：Debug、可观测性与稳定化

> 版本主题：调试、回放、配置校验、失败重跑与指标统计  
> 目标：让 V3 引擎从“能跑”升级为“能调、能定位问题、能稳定迭代”。

---

## 1. 背景

V3.1–V3.4 完成后，系统已经具备：

```text
剧情停滞检测
轻量导演干预
剧情阶段锁
章节连续性
角色信念变化
人物状态继承
```

但系统复杂度也会明显上升。

如果没有调试工具，会很难回答：

```text
Agent 为什么做这个动作？
线索为什么没有触发？
PlotArc 为什么没有推进？
Director 为什么干预？
角色为什么没有改变信念？
章节为什么断裂？
两次运行为什么结果不同？
```

V3.5 的目标是：

> 建立一套可观测、可回放、可校验、可重跑的调试体系。

---

## 2. 版本目标

实现：

```text
RunManager
StateInspector
EventReplayService
AgentTraceViewer
ConfigValidator
RetryReplayService
RunDiffService
MetricsCollector
TuningReportGenerator
```

---

## 3. 不做什么

V3.5 暂不做完整 Web 产品。

不做：

```text
复杂前端编辑器
多用户权限
在线协作
商业化发布
大规模 Dashboard
```

V3.5 先做 CLI + 标准化文件产物。

# 7. V3.5.4 ConfigValidator

## 7.1 目标

运行前检查世界配置是否存在结构错误、引用错误、不可达问题、剧情阶段风险。

---

## 7.2 校验类型

```text
Schema 校验
引用完整性校验
地图可达性校验
线索可触发性校验
剧情阶段校验
Agent 可行动性校验
泄露风险校验
```

---

## 7.3 重点校验

```text
location_id 是否唯一
character_id 是否唯一
clue_id 是否唯一
route_id 是否唯一
route.target 是否存在
route.location_id 是否存在
mandatory clue 是否至少有一条可触发 route
truth 级 clue 是否不会在 setup 阶段开放
public_description 是否泄露 hidden fact
```

## 11. DoD

```text
1. 每次 run 都有 run_manifest.json / run_status.json / run_index.json
2. 每 tick 可保存 state snapshot
3. CLI 能查看 run、state、character、location
4. CLI 能按 tick 回放 events
5. CLI 能查看单个 event 的因果链
6. CLI 能查看 Agent 某 tick 的决策 trace
7. ConfigValidator 能检查 schema、引用、可达性、剧情阶段风险
8. strict 模式下 error 会阻止启动
9. 支持从指定 tick 重跑
10. 支持只重生成指定章节
11. 重跑不会覆盖原 run
12. 支持两个 run 的事件、章节、线索发现差异对比
13. 每次 run 结束生成 metrics.json
14. 每次 run 结束生成 tuning_report.md
15. 运行失败时 errors.jsonl 能记录错误上下文
