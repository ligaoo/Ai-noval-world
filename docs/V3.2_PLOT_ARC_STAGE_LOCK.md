# V3.2 开发计划：PlotArcService + 阶段锁

> 版本主题：剧情阶段管理与线索曝光控制  
> 目标：通过 PlotArcService 控制剧情推进阶段，限制不同阶段可曝光的线索，防止剧透和真相过早暴露。

---

## 1. 背景

V3.1 解决了剧情卡住的问题，但还没有解决：

```text
核心真相可能过早被发现
中后期线索可能在第一章出现
章节推进没有阶段感
Director 不知道当前应该推进哪一类线索
NarrativeWriter 可能暗示后期真相
```

V3.2 要引入 PlotArcService，让系统知道：

```text
当前剧情处于哪个阶段
当前阶段允许发现哪些线索
哪些真相现在必须禁止曝光
当前阶段需要完成哪些事件才能推进
```

---

## 2. 版本目标

实现：

```text
PlotArcService
StageLock
ClueLevel
ArcProgress
ForbiddenRevelationCheck
```

系统应具备：

```text
1. 加载 plot_arcs.json
2. 识别当前 active arc 和 current_stage
3. 给 clue 增加 level / allowed_stages
4. 触发线索前检查阶段是否允许
5. 阶段目标完成后推进 stage
6. 给 AgentContext / NarrativeWriter 注入 forbidden_revelations
7. 防止 setup 阶段发现 major/truth 级线索
```

---

## 3. 不做什么

V3.2 不做复杂多主线。

暂不做：

```text
多条 arc 复杂交叉
自动生成剧情弧
LLM 自主规划长期剧情
复杂情节图谱
```

V3.2 先支持：

```text
单 active arc
线性 stage
按 clue level 控制曝光
```

---

## 4. 核心概念

### 4.1 PlotArc

一条剧情主线。

例如：

```text
旧医院真相篇
失踪妹妹篇
反派阴谋篇
```

V3.2 MVP 先只支持一条 active arc。

---

### 4.2 Stage

剧情阶段。

推荐默认阶段：

```text
setup：建立异常
investigation：调查推进
confrontation：正面冲突
revelation：真相揭露
aftermath：后果处理，可选
```

---

### 4.3 Clue Level

线索等级。

```text
surface：表层异常
minor：轻量线索
medium：中级线索
major：重大线索
truth：核心真相
```

阶段与线索等级默认关系：

```text
setup：surface, minor
investigation：surface, minor, medium
confrontation：surface, minor, medium, major
revelation：surface, minor, medium, major, truth
```

## 6. 模块设计

## 6.1 PlotArcService

### 职责

```text
1. 加载 plot_arcs.json
2. 获取 active arc
3. 获取 current_stage
4. 判断 clue 是否允许在当前阶段发现
5. 判断 stage 是否完成
6. 推进到下一 stage
7. 输出当前阶段上下文给 AgentContext / Director / NarrativeWriter
```

### 核心方法

```python
class PlotArcService:
    def load_arcs(self, world_id: str) -> list[PlotArc]:
        pass

    def get_active_arc(self, simulation_id: str) -> PlotArc:
        pass

    def get_current_stage(self, simulation_id: str) -> PlotStage:
        pass

    def can_discover_clue(self, clue: Clue, arc_state: ArcState) -> bool:
        pass

    def update_progress_from_event(self, event: EventLog) -> ArcState:
        pass

    def maybe_advance_stage(self, arc_state: ArcState) -> ArcState:
        pass

    def get_forbidden_revelations(self, arc_state: ArcState) -> list[str]:
        pass
```

---

## 6.2 StageLock

EnvironmentEngine 在触发线索前调用：

```python
allowed = plot_arc_service.can_discover_clue(clue, arc_state)

if not allowed:
    return blocked_result()
```

blocked_result 不应该暴露系统原因。

示例：

```json
{
  "valid": true,
  "success": false,
  "result": "你翻到了一些旧资料，但纸页受潮严重，暂时无法辨认关键信息。",
  "blocked_reason": "clue_not_allowed_in_current_stage",
  "discovered_facts": []
}
```

## 7. AgentContext 增强

```json
{
  "plot_context": {
    "active_arc": "arc_hospital_truth",
    "arc_stage": "investigation",
    "chapter_goal": "寻找旧医院近期有人出入的证据",
    "allowed_clue_levels": ["surface", "minor", "medium"],
    "forbidden_revelations": [
      "truth_real_killer"
    ],
    "stage_purpose": "收集旧案线索"
  }
}
```

注意：

```text
AgentContext 中 forbidden_revelations 是约束
不要把核心真相 content 明文塞给人物 Agent
可以只塞 revelation_id 或自然语言禁令
```

---

## 8. NarrativeWriter 约束增强

章节生成时输入：

```json
{
  "arc_stage": "investigation",
  "allowed_clue_levels": ["surface", "minor", "medium"],
  "forbidden_revelations": [
    "不能暗示真正凶手身份",
    "不能说明林舟十年前事故责任"
  ]
}
```

NarrativeWriter 必须遵守：

```text
不能提前暗示 forbidden_revelations
不能把 blocked clue 写成已发现事实
不能使用 truth 级线索作为章节钩子
```

---

## 9. Stage 推进规则

stage 完成条件可以由以下组成：

```text
required_clues 全部发现
required_events 全部发生
progress >= threshold
manual_override
```

推荐 MVP：

```text
required_clues 满足即可推进
```

示例：

```python
if all(clue in discovered_clues for clue in stage.required_clues):
    advance_to_next_stage()
```

## 11. 验收用例

### 用例 1：setup 阶段禁止 major 线索

条件：

```text
current_stage = setup
clue.level = major
```

预期：

```text
EnvironmentEngine 不触发 discovered_fact
返回模糊失败结果
写入 blocked_discovery event
```

---

### 用例 2：setup 阶段允许 minor 线索

条件：

```text
current_stage = setup
clue.level = minor
```

预期：

```text
可正常触发 discovery
ArcState progress 增加
```

---

### 用例 3：required_clues 完成后推进 stage

条件：

```text
setup.required_clues = [hf_001]
hf_001 已发现
```

预期：

```text
current_stage 从 setup 推进到 investigation
```

---

### 用例 4：NarrativeWriter 禁止暗示 truth

条件：

```text
forbidden_revelations 包含 real_killer
```

预期：

```text
章节正文不能出现凶手身份暗示
ConsistencyCheck 能拦截
```

## 13. DoD

```text
1. plot_arcs.json 可加载并校验
2. clue 支持 level / arc_id / allowed_stages
3. EnvironmentEngine 触发线索前执行 StageLock
4. setup 阶段不能发现 major/truth 级线索
5. required_clues 满足后 stage 自动推进
6. blocked_discovery 会写入 EventLog
7. AgentContext 能注入当前 arc_stage
8. NarrativeWriter 能收到 forbidden_revelations
9. ConsistencyCheck 能拦截 forbidden_revelation 泄露
10. ArcState 可保存到 state.json
```
