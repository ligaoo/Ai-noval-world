# V3.6 Patch 开发计划：可观测性补丁与版本对齐

> 版本主题：补齐 V3.5 Debug 稳定化的最小闭环，并修复 V1-V3.4 实现与文档之间的关键偏差。  
> 目标：让项目从“主要功能能跑”提升为“每次运行有索引、有状态、有快照、有指标，核心上下文真的进入 Agent 和 NarrativeWriter”。

---

## 1. 背景

截至 V3.5 检查后，项目已经具备：

```text
基础模拟闭环
多地点 move
轻量记忆
LLM 章节改写
导演干预雏形
PlotArc 阶段锁雏形
章节连续性雏形
人物弧光雏形
```

但存在几个阻碍继续迭代的问题：

```text
V3.5 Debug 稳定化产物缺失
代码注释中 V3.4 / V3.5 版本标注混乱
CLI temperature / max_retries 没有真正传入 Agent
V3 上下文字段没有完整进入 Agent prompt
Consistency RuleCheck 基本为空
章节连续性和人物弧光只有状态文件，缺少可调试产物
```

V3.6 Patch 不新增大型玩法系统，优先补齐工程地基。

---

## 2. 版本目标

实现以下最小补丁：

```text
RunManagerLite
state_snapshots
metrics.json
tuning_report.md
errors.jsonl
AgentContext V3 字段注入
CLI 参数真实生效
Consistency RuleCheck 基础拦截

```

## 3. 输出产物

每次 run 至少生成：

```text
outputs/sim_xxx/
  run_manifest.json
  run_status.json
  run_index.json
  state.json
  state_snapshots/
    tick_0001.json
  events.jsonl
  memories.jsonl
  plot_arc_state.json
  character_arcs_state.json
  chapter_plan.json
  chapter_draft.md
  consistency_report.json
  metrics.json
  tuning_report.md
  errors.jsonl
```

说明：

```text
run_manifest.json：记录世界、seed、版本、输入文件和功能开关
run_status.json：记录 running/completed/failed、当前 tick、最后错误
run_index.json：记录关键产物相对路径
state_snapshots：开发期每 tick 保存一次状态快照
metrics.json：运行结束后的基础统计
tuning_report.md：根据 metrics 生成可读调参建议
errors.jsonl：统一追加运行异常上下文
```

## 4. 修复点

### 4.1 CLI 参数生效

`--temperature` 和 `--max-retries` 必须传入 `CharacterAgentService`，不能在 service 内固定为常量。

### 4.2 AgentContext 注入 V3 字段

Agent prompt 至少包含：

```text
plot_arc_stage
plot_arc_purpose
forbidden_revelations
previous_chapter_summary
open_threads
next_chapter_seeds
character_arc_stage
internal_conflict
recent_reflections
soft_director_pressure
```

这些字段是 V3 系统影响角色行为的主要入口。

### 4.3 RuleCheck 基础拦截

`ConsistencyService._rule_check` 至少检查：

```text
出现未访问地点名
出现未发现 clue 的核心 content
出现配置外对象名或角色名的高风险新增实体
```

V3.6 Patch 先采用保守的规则检查；后续再升级为别名表和分词。

### 4.4 版本命名对齐

文档版本以当前 docs 为准：

```text
V3.1：ProgressMonitor + Lightweight Director
V3.2：PlotArcService + 阶段锁
V3.3：ChapterContinuityService
V3.4：Lightweight CharacterArcService
V3.5：Debug / 可观测性 / 稳定化
V3.6：补丁版，补齐 V3.5 最小可观测闭环
```

## 6. DoD

```text
1. 每次 run 生成 run_manifest.json / run_status.json / run_index.json
2. 每 tick 至少保存一个 state snapshot
3. run 失败时 errors.jsonl 追加错误上下文
4. 每次 run 结束生成 metrics.json
5. 每次 run 结束生成 tuning_report.md
6. --temperature / --max-retries 对 LLM Agent 实际生效
7. V3 上下文进入 Agent prompt
8. RuleCheck 不再是空实现
9. README 列出 V3.6 Patch 当前产物
10. 版本注释不再把 CharacterArc 写成 V3.5
```
