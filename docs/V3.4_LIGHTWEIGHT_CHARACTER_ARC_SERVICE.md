# V3.4 开发计划：Lightweight CharacterArcService

> 版本主题：角色信念变化与轻量人物弧光推进  
> 目标：让角色能随着事件改变信念、关系和当前意图，避免多章生成中角色像工具人或每章失忆。

---

## 1. 背景

V3.3 已经保证章节之间的事实和悬念可以继承，但人物仍可能出现：

```text
上一章怀疑看门人，下一章又像没怀疑过
发现重要线索后目标不变化
关系冲突没有长期影响
角色行为只受当前地点影响，不受经历影响
人物心理变化全靠 NarrativeWriter 临时编
```

V3.4 要实现轻量 CharacterArcService。

注意：

> V3.4 不做完整人物弧光系统，先做 belief / relationship / current_intention 三件事。

---

## 2. 版本目标

实现：

```text
BeliefState
RelationshipState
CurrentIntention
CharacterArcLite
ReflectionLite
```

系统应支持：

```text
1. 从事件中更新角色 belief
2. 从对话/冲突中更新 relationship
3. 从章节摘要中更新 current_intention
4. 将角色变化注入 AgentContext
5. 将角色变化提供给 NarrativeWriter
6. 让角色跨章节保持一致心理和行动倾向
```

---

## 3. 不做什么

V3.4 暂不实现完整人物弧光字段：

```text
wound
false_belief
desire
need
acceptance
完整四阶段人物成长
复杂心理模型
```

这些可以放到 V4 或 V3.6+。

V3.4 只做轻量版本：

```text
beliefs
relationships
current_intention
mental_state
```

## 5. 模块设计

## 5.1 CharacterArcService

### 职责

```text
1. 根据 EventLog 更新 beliefs
2. 根据 conflict/dialogue 更新 relationships
3. 根据 chapter_summary 更新 current_intention
4. 生成 ReflectionLite
5. 给 AgentContext 注入角色状态
6. 给 NarrativeWriter 注入人物心理变化依据
```

### 核心方法

```python
class CharacterArcService:
    def update_from_event(self, event: EventLog, state: WorldState) -> CharacterRuntimeState:
        pass

    def update_from_chapter_summary(self, summary: ChapterSummary, state: WorldState) -> CharacterRuntimeState:
        pass

    def generate_reflection_lite(self, agent_id: str, events: list[EventLog]) -> ReflectionLite:
        pass

    def apply_reflection(self, reflection: ReflectionLite, state: WorldState) -> WorldState:
        pass

    def build_character_context(self, agent_id: str, state: WorldState) -> dict:
        pass
```

## 6. 更新规则

## 6.1 belief 更新

事件类型触发：

```text
discovery → 新增或确认 belief
contradiction → 削弱 belief
npc_reveal → 新增 belief
blocked_discovery → 可能新增 suspicion
```

示例：

```text
发现锁最近更换过
↓
新增 belief：旧医院近期有人进入过
confidence = 0.7
```

如果后续发现维修单：

```text
belief strengthened
confidence = 0.9
```

---

## 6.2 relationship 更新

事件类型触发：

```text
npc_refuse_answer → trust -10
npc_lie_detected → trust -20, attitude -10
npc_helpful_answer → trust +10
conflict → attitude -10
threat → fear +15
```

---

## 6.3 current_intention 更新

来源：

```text
chapter_summary.next_chapter_seeds
reflection.next_intention
important discovery
PlotArc current_stage
```

优先级：

```text
manual override > reflection > chapter_summary seed > plot_arc stage goal
```

## 7. AgentContext 增强

```json
{
  "character_state": {
    "mental_state": "uneasy_and_suspicious",
    "active_beliefs": [
      "旧医院近期有人进入过"
    ],
    "relationships": {
      "char_guard": {
        "attitude": -25,
        "trust": 20,
        "reason": "看门人多次回避关键问题"
      }
    },
    "current_intention": "寻找近期进入医院的人是谁"
  }
}
```

Agent 决策时应遵守：

```text
行动应符合 current_intention
对 NPC 的态度应受 relationship 影响
不能忽略 active_beliefs
```

## 8. NarrativeWriter 输入增强

```json
{
  "character_arc_context": {
    "pov_character": "char_linzho",
    "mental_state": "uneasy_and_suspicious",
    "belief_changes": [
      "从认为医院只是梦境地点，变为怀疑现实中有人近期进入过"
    ],
    "relationship_changes": [
      "对看门人的信任下降"
    ],
    "current_intention": "继续追查近期出入者"
  }
}
```

NarrativeWriter 可以增强心理描写，但不能新增事实。

## 9. 触发 ReflectionLite 的时机

```text
章节结束
重要线索发现
重大冲突事件
关系值变化超过阈值
角色 current_intention 为空
```

默认：

```text
每章结束生成一次 ReflectionLite
```

## 10. 验收用例

### 用例 1：发现线索更新 belief

条件：

```text
event_type = discovery
clue = hf_001 锁最近换过
```

预期：

```text
char_linzho 新增 belief：旧医院近期有人进入过
confidence >= 0.6
```

---

### 用例 2：NPC 回避降低 trust

条件：

```text
char_guard 拒绝回答 recent_visitors
```

预期：

```text
char_linzho.relationships.char_guard.trust 降低
```

---

### 用例 3：章节结束更新 intention

条件：

```text
chapter_summary.next_chapter_seeds 包含“追问锁的来源”
```

预期：

```text
current_intention 更新为“追查锁的来源”
```

---

### 用例 4：下一章 AgentContext 保留心理状态

条件：

```text
ch_001 结束后 char_linzho suspicious
```

预期：

```text
ch_002 AgentContext 包含 mental_state 和 active_beliefs
```

## 12. DoD

```text
1. discovery event 能新增或更新 belief
2. dialogue/conflict event 能更新 relationship
3. chapter_summary 能更新 current_intention
4. 每章结束能生成 ReflectionLite
5. AgentContext 能注入 beliefs / relationships / current_intention
6. NarrativeWriter 能获取 character_arc_context
7. 下一章角色不会丢失上一章的怀疑和关系变化
8. belief 支持 strengthened / confirmed / rejected
9. relationship 变化有 source_event_id
10. current_intention 可追溯来源
```
