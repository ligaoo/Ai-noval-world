# Novel Simulator V5.2 - 启动说明

## 🚀 快速启动

### 方式一：Python 启动（推荐）

```bash
python launcher.py
```

### 方式二：PowerShell 启动

```powershell
.\start.ps1
```

### 方式三：启动菜单（Windows 推荐）

双击 `启动菜单.bat` 打开交互式启动菜单，可选择启动方式、检查环境、安装依赖。

## 📋 服务说明

启动后将运行两个服务：

| 服务 | 端口 | 地址 | 说明 |
|------|------|------|------|
| 后端 API | 8421 | http://localhost:8421 | FastAPI 服务器 |
| 前端 Web | 4242 | http://localhost:4242 | Vue3 + Vite 开发服务器 |

### API 文档

- **Swagger UI**: http://localhost:8421/docs
- **ReDoc**: http://localhost:8421/redoc

## 📦 首次运行准备

### 1. 环境要求

- **Python**: 3.10+
- **Node.js**: 16+

### 2. 安装依赖（首次运行自动执行）

Python 依赖会自动安装：
```bash
pip install -r requirements.txt
```

前端依赖会自动安装：
```bash
cd web
npm install
```

## 🔧 手动启动（开发调试）

如果需要单独启动某个服务进行调试：

### 启动后端 API

```bash
# 方式一
python -m uvicorn api.server:app --host 0.0.0.0 --port 8421 --reload

# 方式二
cd api
python server.py
```

### 启动前端

```bash
cd web
npm run dev
```

## 📡 API 接口列表

### 世界配置
- `GET /api/worlds` - 获取所有世界列表
- `GET /api/worlds/{world_id}` - 获取世界详细配置

### 模拟运行
- `GET /api/simulations` - 获取所有模拟记录
- `GET /api/simulations/{sim_id}` - 获取模拟详情
- `GET /api/simulations/{sim_id}/quality` - 获取模拟质量报告
- `POST /api/simulations/run` - 运行新模拟

### 类型系统
- `GET /api/genres` - 获取所有可用类型
- `GET /api/genres/{genre_id}/profile` - 获取类型配置文件

## 🎯 V5.2 新功能

### Genre Abstraction Layer 类型抽象层
- 支持多种故事类型
- 可扩展的类型包系统
- 通用基础类型 (Generic)
- 恐怖灵异类型包 (Horror)

### Horror Genre Pack 恐怖类型包
- 4 个递进阶段：
  - **subtle_anomaly** (轻微异常)
  - **clear_threat** (威胁显现)
  - **rule_discovery** (规则探索)
  - **truth_and_resolution** (真相揭示)
- 20 种恐怖手法
- 类型专用质量评估维度
- 恐怖强度递进控制

### Story Quality Evaluator 质量评估器
- base_scores (通用维度评分)
- genre_scores (类型专项评分)
- weighted_overall (加权总分)
- 自动修稿建议
- 质量趋势追踪

## ⚙️ 配置说明

### 类型选择

修改 `launcher.py` 或通过 API 调用时指定 `genre_id`：

```python
# 支持的类型
genre_id = "horror"    # 恐怖灵异
genre_id = "generic"   # 通用故事
```

### 小说长度设置

```python
target_chapters = 10  # 总章节数，控制恐怖强度曲线
chapter_no = 1       # 当前章节号
```

## 🛠️ 故障排除

### 端口被占用

修改端口：
- 后端：`api/server.py` 中的 `port=8421`
- 前端：`web/vite.config.js` 中的 `server.port`

### 前端启动失败

```bash
cd web
rm -rf node_modules package-lock.json
npm install
npm run dev
```

### 后端导入错误

确保 `PYTHONPATH` 包含项目根目录：
```bash
set PYTHONPATH=%CD%
python -m uvicorn api.server:app --port 8421
```

### 依赖安装失败

```bash
# 升级 pip
python -m pip install --upgrade pip

# 重新安装
pip install -r requirements.txt
```
